Для демонстрации работы политики ingress было установлено дополнительное приложение app2, возвращающее информацию о запросе в формате json.
В качестве ingress-контроллера был использован Application Load Balancer, совместимый с Managed Service for Kubernetes от Yandex Cloud, который я использую.
В values можно увидеть значения для ingress, специфичные для этого контроллера.
Секрет для доступа к приватному репозиторию был создан командой kubectl create secret docker-registry, он работает в рамках обозначенного namespace.
В шаблонах вы не увидите объявления namespace, оно было создано при применении чарта.
